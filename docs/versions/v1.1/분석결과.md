#### #8 포인트 시스템이 어려운 이유

**신규 DB 테이블 3개:**
- `points` - 유저별 현재 포인트 잔액
- `point_transactions` - 적립/차감 내역 (출석 +10, 글쓰기 +5 등 전부 기록)
- `point_rules` - 관리자가 행동별 점수 설정 (action_type, points, daily_limit)

**핵심 난이도:**
- 동시성 문제: 동시 추천 시 포인트 꼬임 방지 → DB 트랜잭션 필수
- 중복 방지: 출석 1일 1회, 같은 글 추천 1회 → 중복 체크 로직
- 기존 코드 수정: 글쓰기/추천/댓글 등 기존 기능에 포인트 적립 로직 삽입
- 관리자 설정 페이지: 행동별 점수 CRUD + 실시간 반영
- 마이페이지: 포인트 이력 조회 UI

#### #9 도박 시스템이 어려운 이유

**배팅:**
- 관리자가 배팅 주제 생성 (선택지 설정)
- 유저 포인트 배팅 → 마감 → 결과 선택 → 배당금 자동 분배
- 배당률 계산 (총 풀 / 정답자 수), 소수점/반올림/0원 엣지케이스

**상자 뽑기:**
- 100p 투입 → 결과: 0, 5, 10, 100, 200p
- 서버 사이드 가중 랜덤 (클라이언트 조작 방지)
- 관리자 확률 조정 기능
- 뽑기 애니메이션 UI

**공통:**
- 부정행위 방지 → 서버 사이드 검증 필수
- 동시 접속 시 트랜잭션 처리 (100명 동시 뽑기)

#### #16 퍼스나콘이 어려운 이유
